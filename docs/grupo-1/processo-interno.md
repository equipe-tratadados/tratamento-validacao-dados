# Processo Interno — Grupo 1

## 1. Objetivo

Definir o fluxo mínimo de gestão do Grupo 1 para garantir previsibilidade, rastreabilidade e entregas compatíveis com o padrão esperado pela coordenação geral.

## 2. Princípios Operacionais

- Toda tarefa precisa ter responsável, prazo e formato de entrega.
- Toda entrega precisa ter evidência verificável: link, arquivo, print ou comentário estruturado.
- Toda fonte precisa ser documentada antes de ser considerada validada.
- Todo bloqueio precisa ser comunicado antes do vencimento do prazo.
- Toda decisão relevante deve ser registrada.

## 3. Fluxo de Solicitação

```txt
Grupo de líderes / Coordenação
        ↓
Líder do Grupo 1
        ↓
Tradução da solicitação em tarefa objetiva
        ↓
Distribuição ao Grupo 1
        ↓
Confirmação dos membros
        ↓
Execução
        ↓
Entrega
        ↓
Validação pelo líder
        ↓
Reporte ao grupo de líderes
```

## 4. Formato de Solicitação Interna

Toda solicitação enviada ao Grupo 1 deve seguir o formato:

```txt
@Nome ou @Grupo 1 — [pedido objetivo].
Prazo: [data/hora].
Formato de resposta: [link / print / CSV / comentário estruturado].
Critério de aceite: [o que será considerado entrega válida].
```

## 5. Formato de Resposta dos Membros

```txt
@Nome — Status: [recebido / em andamento / bloqueado / entregue].
Prazo: [data/hora].
Formato de entrega: [link / print / arquivo / comentário].
Bloqueio: [se houver].
```

## 6. Estados de Tarefa

| Status | Significado |
|---|---|
| Backlog | Tarefa identificada, mas ainda não iniciada |
| A fazer | Tarefa pronta para execução |
| Em andamento | Tarefa assumida por responsável |
| Em validação | Entrega enviada e aguardando revisão |
| Bloqueada | Impedimento declarado antes do prazo |
| Concluída | Entrega validada pelo líder |

## 7. Prazos

| Tipo de prazo | Regra |
|---|---|
| Confirmação de recebimento | A definir pelo grupo: 24h, 48h ou 72h |
| Primeira resposta estruturada | A definir pelo grupo |
| Entrega técnica | Definida conforme complexidade da tarefa |
| Comunicação de bloqueio | Sempre antes do vencimento |

## 8. Validação Interna

Uma entrega só será considerada válida quando tiver:

- responsável identificado;
- arquivo, link ou evidência anexada;
- fonte documentada, quando aplicável;
- status claro;
- ausência de bloqueio não declarado;
- conformidade com o padrão definido para CSVs e documentação.

## 9. Registro

As decisões e entregas devem ser registradas em pelo menos um dos canais formais:

- GitHub Project;
- issue ou pull request no GitHub;
- página do Grupo 1 no Notion;
- ata da daily.
